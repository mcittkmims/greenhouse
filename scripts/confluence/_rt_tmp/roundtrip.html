SHR-13

GMS-22

User Story

Farmer

Must Have

Configure time-based irrigation schedules (start time, duration, frequency) that execute autonomously without requiring sensor input, cloud connectivity, or on-site presence

When the farmer is absent from the greenhouse for an extended period — ranging from days to several months — or when sensor-based control is temporarily unavailable due to hardware failure or connectivity loss

Irrigation continues on a predictable schedule regardless of absence duration or connectivity status; crops are maintained autonomously; the greenhouse remains fully operational without anyone on-site

From WP4.2: one survey respondent required fully autonomous operation for up to 3 months while abroad; schedule-based irrigation ensures continuity when connectivity or sensor feedback is unavailable — this is the fallback mechanism when real-time sensor control (GMS-16) is unavailable. Critical requirement: timed schedule must co-exist with and not conflict with threshold-based sensor-triggered irrigation (GMS-16). Conflict resolution rule: a scheduled irrigation run should be automatically skipped or aborted if soil moisture is already at or above the target setpoint at execution time — this prevents over-watering when the soil is already wet. The edge controller must store and execute schedules locally without requiring cloud connection — hard offline autonomy requirement per Project Charter O5 (72h offline minimum; the 3-month use case requires indefinite offline schedule execution). Schedule management UI must support: create, edit, delete, and view upcoming scheduled runs from dashboard. Schedule execution events must be logged with timestamp, planned time, actual start time, and skip reason (if skipped). Acceptance: scheduled irrigation runs at configured time during cloud outage; skips if soil already at or above setpoint; operator can fully manage schedules from dashboard; execution logged. Cross-references: GMS-28 (Job Story pair, SHR-14), GMS-16 (sensor-triggered control — co-existence required), GMS-26/32 (auto-stop applies to scheduled runs too), GMS-23/29 (configurable setpoints used for skip-if-wet logic).

PROPOSEDYellow

Requirement analyzed and refined. Decomposed into: User = farmer absent for extended periods (days to months), Action = configure and execute autonomous timed irrigation schedules, Circumstances = extended absence or sensor/connectivity failure making real-time control unavailable, Outcome = crops maintained autonomously without on-site presence. Confirmed Must Have — survey evidence of 3-month autonomous operation requirement; also serves as fallback when GMS-16 sensor-based control is unavailable. Key clarifications: (1) schedule must not conflict with sensor-triggered control — skip if soil already at setpoint (prevents over-watering); (2) edge controller executes schedules locally without cloud (Project Charter O5); (3) auto-stop (GMS-26/32) still applies during scheduled runs. Full schedule management (create/edit/delete) from dashboard required; execution events logged. Linked to GMS-28 (JS pair, SHR-14). Moving to Proposed.
